## 1. 개요

파일럿에서 확인된 핵심 문제 — *"계속 어려워지는 Adaptive 시험"* 으로 인한 부정확한 레벨 배정 — 을 해결하기 위해, **교재 범위 안에서 정확히 진단하는 구조**로 전환한다. 기존 자체 제작 10종 지필 시험지(레벨별 30문항 + 진단표)를 온라인 문항 데이터로 활용한다.

---

## 2. ✅ 확정 사항 (실무진 컨펌 완료)

| # | 항목 | 확정 내용 |
|---|---|---|
| 1 | 자가진단 화면 | **완전 제거.** 학생 자가진단 플로우를 없애고, "원장 배정 → 학생 응시"로 단순화 |
| 2 | 90점+ 보너스 문제 | **상위 레벨 시험지에서 문항을 추출**해 구성 (별도 보너스 문항 제작 없음) |
| 3 | STT 텍스트 시각화 | **학생에게 미노출, 원장에게만 표시** (원장 판단 자료로 활용) |
| 4 | Writing 레벨별 차등 | **Seeker 제외, 전 레벨 청크(덩어리) 단위 Drag & Drop으로 유형 통일.** 유형은 하나, 난이도(청크 수·문장 길이·어휘)만 레벨별 차등 |

> **참고 — "매번 랜덤 추출" 풀**: 70:30 구조(선택 레벨 + 상위 인접 레벨 합산)로 출제 풀이 30문항보다 커지므로, 기존 10종 문항만으로 재응시 시 동일 문제 방지가 가능할 것으로 본다. Phase 0 현황 진단에서 최종 확인.
> **개발 측 정리 사항(추가 컨펌 불필요)**: 보너스 문항 추출 시 최상위 Innovator는 상위 레벨이 없으므로, Innovator 응시자는 보너스를 생략하거나 동일 레벨 상위 세트(7–12)에서 추출 — 출제 로직 단계에서 확정.

## 3. 개선 요청 → 작업 매핑

| # | 영역 | 요청 내용 |
|---|---|---|
| ① | 난이도·문항 구성 | 70:30 구성 / 90점+ 보너스(상위 레벨 추출, 화면 명시) / 매번 랜덤 추출(재응시 동일문제 방지) |
| ② | 시험지 배정 | 원장이 관리자 페이지에서 배정 + 시험지별 가이드 제공 / 자가진단 화면 제거 |
| ③ | 라이팅 | 전 레벨(Seeker 제외) 청크 단위 Drag & Drop 통일, 난이도만 차등 |
| ④ | 스피킹 | 레벨 결정 제외·보조지표화 / 음성 응답률 리포트 수치 표시 / STT는 원장만 노출 / 응시환경 주의 문구 / 음성 응답 원칙+예외 타이핑 안내 / 의미 일치 시 다양한 답변 수용 확대 |
| ⑤ | 결과 리포트 | 영역별 그래프·표 / 적정 레벨 + 도전 레벨 동시 제시 / 경쟁사 비교표(원장 상담 참고자료) |
| ⑥ | 관리자 | 이메일 필수값 / 원장 로그인 시 본인 학원 학생만 조회 |
| ⑦ | 리스닝 | Speaking을 'Listen & Speak'(듣고 말한다) 형태로 운영해 리스닝 요소 반영 |

## 4. 작업 방식 & 순서

기능이 서로 맞물려 있어 **기반부터 쌓고 → 결과 리포트를 마지막에** 붙인다. (원장 배정은 권한 설정이 선행되어야 하고, 리포트는 채점·스피킹 데이터가 나온 뒤 가능.)

진행 방식: 각 Phase를 클로드 코드 프롬프트로 단계별 실행. 코드는 `englab-ai-test` repo 기준.

### Phase 0 — 현황 진단 & 데이터 설계 (기반)
- 읽기 전용 감사: repo 구조·데이터 모델·각 기능 현재 동작 파악 → `docs/현황진단_0629.md` 산출
- 문항 은행(item bank) 스키마 설계, 10종 시험지·정답키·진단표 적재 계획
- 랜덤 추출 풀 충분 여부 최종 판단
- *의존성: 없음 (즉시 착수)*

### Phase 1 — 관리자 인증 & 권한 (⑥)
- 이메일 필수값 전환 (현재 옵션 → 누락 발생)
- 원장 계정 로그인 시 **본인 학원 학생만** 조회되도록 권한 설정
- *의존성: P0*

### Phase 2 — 시험지 배정 + 자가진단 제거 (②)
- 원장이 관리자 페이지에서 학생에게 시험지 배정 + 시험지별 가이드 표시
- **기존 학생용 자가진단(경험 체크) 화면·로직 완전 제거**
- 학생 플로우: 배정받은 시험지 응시
- *의존성: P0(시험지 세트 정의) + P1(권한)*

### Phase 3 — 출제 로직 (①)
- 70:30 구성(선택 레벨 70% + 상위 인접 30%)
- 매번 랜덤 추출(재응시 동일문제 방지)
- 90점+ 보너스: **상위 레벨 시험지에서 추출**, 화면에 "보너스 문제" 명시
- *의존성: P0(태깅된 item bank)*

### Phase 4 — 라이팅 입력 방식 (③)
- 전 레벨(Seeker 제외) **청크 단위 Drag & Drop 단일 유형**으로 통일
- 레벨별 차등은 유형이 아니라 **난이도**(청크 개수·문장 길이·어휘 수준)로만
- *의존성: P0(쓰기 문항을 청크 데이터로 모델링)*

### Phase 5 — 스피킹 & Listen & Speak (④⑦)
- 스피킹 → 레벨 결정 제외, **보조지표화**
- **Listen & Speak**: AI 음성 질문을 듣고 답하는 구조로 리스닝 요소 반영
- 음성 응답 비율 집계(→ 리포트 수치 표시)
- 의미 일치 시 다양한 답변 수용 확대(STT 정확도·유사답변 허용)
- 음성 응답 원칙 + 예외 타이핑 안내, 응시환경 주의 문구
- **STT 텍스트는 원장에게만 노출**(학생 화면 미표시)
- *의존성: P0*

### Phase 6 — 결과 리포트 (⑤)
- 영역별 그래프·표(직관적), 단계/레벨/CEFR 매핑
- **적정 레벨 + 도전 레벨 동시 제시**
- 스피킹 음성 응답률 수치 표기, STT(원장 뷰)
- 진단표 연동(점수 → 추천 교재 단계 + 진단 문구)
- 경쟁사 레벨 비교표(별도 제작 → 원장 상담 참고자료)
- *의존성: P3(채점)·P5(스피킹 지표)*

### Phase 7 — 통합 QA & 배포
- 전 기능 통합 검수 → 직영점 재파일럿 → 최종 보정 → 정식 배포
- *의존성: P1~P6*

### 의존성 흐름
```
P0(데이터·진단)
  ├─ P1(권한) ── P2(배정·자가진단제거)
  ├─ P3(출제로직) ─┐
  ├─ P4(라이팅)    ├─ P6(리포트) ── P7(QA·배포)
  └─ P5(스피킹)  ──┘
```
